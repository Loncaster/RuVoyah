# Contributing

## Общий порядок работы

1. Найдите нужное штатное приложение на автомобиле.
2. Извлеките штатный APK с автомобиля.
3. Достаньте из APK исходные строки.
4. Найдите соответствующий overlay-проект в [`source code`](source%20code).
5. Добавьте или исправьте перевод.
6. Сделайте fork репозитория и откройте pull request в `main`.
7. В комментарии к pull request напишите `/build`.
8. Скачайте preview-пакет из workflow и проверьте его на автомобиле.
9. Отметьте checklist в pull request.

## Правила перевода

- Новый текст по возможности должен быть не длиннее английского или китайского оригинала.
- Если перевод длиннее оригинала, его обязательно нужно проверить на реальном автомобиле до отправки pull request.
- Если в PR есть строки длиннее оригинала, в описании PR нужно явно написать, что они были проверены на автомобиле.
- Старайтесь использовать одну и ту же терминологию в разных overlay-проектах.

## Где вносить изменения

- Переводы лежат в `source code/<project>/app/src/main/res/values*/strings.xml`.
- Переназначение строк на штатное приложение задаётся в `source code/<project>/app/src/main/res/xml/overlays.xml`.
- Изменяйте только тот overlay-проект, который соответствует целевому штатному пакету.

## Проверка изменений

- Минимум: убедиться, что проект собирается.
- Если вы меняли строки с риском переполнения интерфейса, обязательно проверьте их на автомобиле.
- Проверка на автомобиле для переводческих PR ожидается до финального review.

## Preview-пакеты

- Основной публичный репозиторий публикует только стабильные релизы из `main`.
- Для проверки изменений откройте PR из своего fork и оставьте комментарий `/build`.
- Preview-пакет собирается из текущей ветки PR и прикладывается как artifact workflow.
- После успешной сборки бот оставляет в PR ответный комментарий со ссылкой на ZIP-артефакт.
- Такой пакет не попадает в GitHub Releases и нужен только для проверки на автомобиле до merge.

## Pull Request

В PR желательно указать:

- какое приложение переводилось
- какие экраны или сценарии менялись
- были ли строки длиннее оригинала
- были ли эти строки проверены на автомобиле
- собран ли preview-пакет из текущей ветки PR и проверен ли он на автомобиле
