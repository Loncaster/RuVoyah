# Процесс релизов

## Основная модель

- `main` содержит только принятые изменения после review.
- Pull request в `main` проверяются автоматической сборкой.
- После merge в `main` автоматически собирается стабильный релиз.

## Проверка Pull Request

Workflow PR:

- собирает все overlay-проекты
- не публикует APK и не создаёт релиз
- нужен только для проверки, что изменения не ломают сборку

## Preview для переводчиков

Preview существует только в форках переводчиков.

Почему так:

- основной публичный репозиторий должен публиковать только стабильные, проверенные пакеты
- переводчик может безопасно собрать свою ветку в собственном форке и проверить её на автомобиле

Ожидаемый порядок:

1. Сделать fork.
2. Запушить свою ветку.
3. Запустить ручной workflow preview-сборки в форке.
4. Скачать пакет артефактов.
5. Проверить изменения на автомобиле.
6. Открыть pull request в основной репозиторий.

## Стабильный релиз

Workflow стабильного релиза:

- запускается на `push` в `main`
- собирает все overlay APK
- формирует установочный архив
- публикует его в GitHub Releases

## Что входит в стабильный релиз

- все overlay APK
- `adb` и сопутствующие файлы
- install/uninstall scripts
- disable-verity scripts
- краткая инструкция по установке и удалению

## Что не входит в релиз

- штатные APK из автомобиля
- preview-пакеты из форков
- build-артефакты разработчиков
